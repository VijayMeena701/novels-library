# Backlog

This folder captures features, refactors, and architectural decisions that were discussed but deferred. Items here are intentionally not implemented yet; they are kept for review and prioritization.

## Status labels

- **needs-review** — Not yet decided; needs discussion before work starts.
- **deferred** — Agreed to be useful but postponed to a later iteration.
- **ready** — Decision is made; waiting for implementation time.

## Adding items

Create a new markdown file in this folder with a short title, the status, and a description of the problem/decision and the proposed approach.
